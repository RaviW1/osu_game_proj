Features Implemented
Camera System (Camera2D.cs)

Custom MonoGame camera using a transform matrix passed to spriteBatch.Begin
Smooth lerp-based following with configurable LerpSpeed
Room bounds clamping so camera never shows void
SnapTo for instant repositioning on room load
Two-pass draw system Ś world space with camera transform, HUD screen space without

Scene System (IScene.cs, GameScene.cs)

IScene interface with Load, Update, Draw, Unload
Game1 reduced to three method calls Ś owns nothing except the current scene
GameScene owns all game state that previously lived in Game1
Static instance pattern eliminated
CycleEnemy, CycleBlock, CycleStage moved to GameScene as instance methods

Room System (IRoom.cs, RoomBase.cs, RoomA.cs, RoomB.cs)

IRoom interface with Load, Update, Draw, Unload, GetSpawnPoint
RoomBase abstract class owns tiles, spawn points, collision
Each room declares its own Bounds which camera automatically clamps to
TileBlock now carries tileType so tile identity travels with the tile
Geo.PlaceGeosOnPlatforms overload added for List<TileBlock>
TileGenerator still used internally by rooms but no longer exposed to GameScene
CycleStage replaced by room swap Ś camera bounds update automatically

Collision System Rewrite (CollisionSystem.cs, CollisionResult.cs, CollisionDirection.cs)

PhysicsHelper eliminated entirely
CollisionSystem.Query Ś pure detection, returns List<CollisionResult>, knows nothing about game objects
CollisionResult struct carries tile, overlap rectangle, direction, harmful flag, collideable flag
CollisionDirection enum Ś Up, Down, Left, Right
Velocity-informed direction detection Ś uses dominant velocity axis to resolve ambiguous overlaps
Two-phase pattern Ś broadphase query feeds narrowphase resolution
Vertical collisions sorted before horizontal to prevent corner compounding
Player.ResolveCollisions Ś player owns its own collision response
IEnemy.ResolveCollisions Ś enemies own their own collision response
HandleOverlap eliminated

Enemy Architecture (IEnemy.cs)

IEnemy interface added Ś TakeDamage, BounceX, BounceY, GetVelocityX, GetVelocityY, ResolveCollisions
Aspid, Boofly, HuskBully all implement IEnemy
Enemy type-checking collapsed from three separate if/else blocks to one is IEnemy check
New enemies just implement IEnemy and get collision for free

Player Architecture

Gravity moved to Player.Update Ś runs every frame regardless of state
States no longer own physics Ś only animation and input response
Velocity.X used for horizontal movement so collision direction detection works
Invincibility frames Ś invincibilityTimer ticks down in Update, TakeDamage early-exits if invincible
player.Tiles removed Ś room owns tiles now
HandleOverlap removed Ś replaced by ResolveCollisions
Walk(0) calls StopWalking instead of Walk on the current state

State Machine Improvements

IPlayerState.Reset renamed to OnEnter Ś eliminates naming confusion with game reset command
StopWalking added to IPlayerState Ś clean separation of stop vs walk intent
SuppressLandingTransition cleared on enter of all grounded states as defensive safety net
JumpState.OnEnter sets OnGround = false immediately
MovementAxisCommand guards against double-execution per frame when multiple keys bound to same command

Input Architecture

BindKeyboardKeys renamed to BindKeys
CycleStageCommand, CycleEnemyCommand, CycleBlockCommand take GameScene instead of static Game1
QuitCommand still takes Game1 since only Game can call Exit()
ResetCommand takes GameScene
MouseController takes GraphicsDevice instead of Game1


Known Issues / Bandaid Fixes
1px ground overlap (Position.Y -= result.Overlap.Height - 1)
Subtracts 1 pixel less than the full overlap to keep player slightly embedded in ground tile. Ensures ResolveCollisions always finds a Down collision next frame. Proper fix would be a dedicated ground sensor that doesn't rely on pixel-perfect overlap.
Remaining state bugs
Occasional edge cases when rapidly switching between attack and heal states. SuppressLandingTransition defensive clearing in OnEnter mitigates but may not catch every case.
HuskBully.floorY unused warning
floorY field is assigned but death fall logic was never fully implemented for HuskBully.
WalkingState.commandReceivedThisFrame unused
Left over from an earlier approach to input detection, never cleaned up.
